# Sts2Speak — 尖塔2联机聊天 Mod

为《Slay the Spire 2》多人联机局提供一个**局内文字聊天框**。

你可以按 `Tab` 打开聊天输入框，给队友直接打字；队友发言时，屏幕上会短暂显示聊天记录预览；如果当前处于战斗中，聊天内容还会尽量复用游戏原生的**头顶说话气泡**表现。

---

## 简介

《Slay the Spire 2》联机时默认没有直接的打字交流入口。  
`Sts2Speak` 的目标很简单：

- 给联机玩家一个**顺手可用**的聊天框
- 不强行打断操作，不把 UI 做得过重
- 尽量贴近游戏现有表现，尤其是战斗中的说话气泡

当前版本优先解决“**能聊、好用、尽量不挡视野**”这三个问题，不做表情或历史持久化。

---

## 功能

- 按 `Tab` 打开或关闭聊天框
- 聊天框分为：
  - 上方：聊天记录区
  - 下方：输入框 + 发送按钮
- 队友发言时：
  - 只显示聊天记录预览
  - 不强制弹出输入框
- 预览会：
  - 先停留 `5` 秒
  - 再在 `4` 秒内逐渐透明并隐藏
- 渐隐期间如果你按 `Tab`：
  - 会立刻取消渐隐
  - 并回到正常的 `Tab` 显隐逻辑
- 战斗中会尽量复用原生 `NSpeechBubbleVfx`：
  - 让发言内容显示在该玩家头顶
  - 视觉上尽量接近游戏已有的说话气泡

---

## 使用方式

### 打开聊天框

1. 进入一个多人联机局
2. 按 `Tab`
3. 聊天框会显示在界面左下区域
4. 输入消息后按回车，或点击“发送”

### 隐藏聊天框

- 在输入态下再次按 `Tab`，聊天框会直接隐藏

### 队友发言时会发生什么

- 如果你当前**没有主动打开输入框**：
  - 屏幕上只会出现聊天记录预览
  - 不会把输入框硬弹出来
- 预览在 `5` 秒后开始淡出
- 淡出总时长为 `4` 秒
- 期间如果你按 `Tab`：
  - 会中断淡出
  - 并切换到正常输入态

### 战斗中的表现

- 如果当前是战斗场景
- 且游戏能拿到发言玩家对应的 `Creature`
- 那么聊天内容会尽量显示为角色头顶说话气泡

如果当前时机不满足这些条件，也不会报错，消息仍然会正常出现在聊天记录里。

## 安装

尖塔2官方已经支持 Modding，并提供了 Harmony 与相关运行时支持。

### 安装步骤

1. 在游戏目录下创建 `mods` 文件夹
2. 将本 Mod 的以下文件放入该目录：
   - `Sts2Speak.dll`
   - `Sts2Speak.pck`
3. 启动游戏

默认路径示例：

- `E:/SteamLibrary/steamapps/common/Slay the Spire 2/mods`

---

## 关于开启 Mod 的提醒

和其他尖塔2 Mod 一样，开启 Mod 后，游戏通常会弹出一次官方 Mod 提示。

一般需要注意：

- Modded 模式与原版存档是分开的
- Steam 云同步可能会提示冲突
- 如果你非常在意原版存档，请先自行备份

如果你本来就经常玩尖塔2 Mod，这一部分应该已经比较熟悉了；  
如果你第一次装 Mod，建议先确认自己的 `mods` 目录和存档习惯。


---

## 已知限制

- 仅在**真实多人联机局**中生效
- 单人模式或伪联机状态下不会启用聊天功能
- 当前输入框是单行输入
- 消息长度会在服务层做截断
- 头顶说话气泡属于 **best effort**
  - 主要面向战斗场景
  - 如果当前玩家 `Creature` 不可用，就只显示聊天记录

## 开发说明

本项目当前实现基于：

- Godot 4.5.1
- C# / .NET 9
- Harmony
- `sts2.dll`

项目结构参考了已有的 StS2 Mod 写法，核心目录包括：

- `Messages/`
- `Services/`
- `Ui/`
- `Patches/`

当前核心文件：

- `MainFile.cs`
- `Patches/GameBootstrapPatch.cs`
- `Messages/ChatBroadcastMessage.cs`
- `Services/ChatService.cs`
- `Ui/ChatOverlay.cs`

如果你要自行编译，请先按本机环境修改：

- `Sts2Speak.csproj` 中的 `GodotPath`
- `Sts2Speak.csproj` 中的 `Sts2Path`
